# 🏥 نظام حجز مستشفى ميديكير — Medicare Hospital Booking System

نظام حجز مواعيد إلكتروني لمستشفى، بواجهة عربية/إنجليزية قابلة للتبديل، وأربعة ثيمات لونية طبية، وخلفية PHP + MySQL كاملة لحجز المواعيد والتواصل.

## 🚦 التسليم على مرحلتين (اقرأ هذا أولًا)

**المرحلة 1 — تعمل الآن فورًا بدون أي استضافة أو سيرفر:**
كل الموقع (4 صفحات + التصميم الكامل + تبديل اللون واللغة) يعمل بفتح `index.html` مباشرة من أي متصفح، أو برفع مجلد الموقع كما هو (HTML + CSS + JS فقط) على أي استضافة.
تسجيل الدخول، إنشاء حساب مريض، وتسجيل الخروج **يعملون فعليًا الآن** عبر تخزين المتصفح (`localStorage`) دون الحاجة لأي سيرفر أو قاعدة بيانات:
- أنشئ حسابًا من `login.html`، سجّل دخولك، وسترى اسمك يظهر أعلى القائمة مع زر "خروج" في كل صفحات الموقع.
- هذه البيانات محفوظة داخل متصفحك الحالي فقط، وهذا طبيعي في هذه المرحلة.

**المرحلة 2 — تُفعَّل عند تجهيز الاستضافة:**
نموذجا "حجز موعد" و"تواصل معنا" مصمَّمان بالكامل ليعملا مع خلفية PHP + MySQL جاهزة تمامًا في مجلد `php/`. حاليًا -بدون سيرفر- ستظهر رسالة "تعذر الاتصال بالخادم" عند إرسالهما، وهذا متوقع تمامًا لحين رفع الملفات على استضافة تدعم PHP واستيراد `php/database.sql`. عند ذلك يعمل حجز المواعيد وإرسال الرسائل فعليًا مع تخزينها في قاعدة البيانات.

## ✨ المزايا المتوفرة حاليًا

- 4 صفحات HTML منفصلة ومترابطة بتنقل موحّد.
- تصميم طبي عصري وموثوق (ثيم أزرق طبي أساسي) مع 3 ثيمات إضافية (أخضر Care Green، بنفسجي Violet Care، داكن) يبدّلها الزائر من أيقونات الألوان في القائمة، ويُحفظ اختياره في المتصفح.
- زر واحد لتبديل اللغة بين العربية (RTL) والإنجليزية (LTR) لكل عناصر الصفحة دفعة واحدة، مع حفظ اللغة المختارة.
- تسجيل دخول + إنشاء حساب + **خروج** يعمل بالكامل من طرف المتصفح (localStorage)، مع تحقق كامل من صحة المدخلات (بريد، جوال، رقم هوية، تطابق كلمة المرور). شارة اسم المريض وزر الخروج يظهران تلقائيًا في قائمة كل الصفحات بعد تسجيل الدخول.
- نموذج حجز موعد من 3 خطوات: اختيار القسم الطبي (يُحدّث قائمة الأطباء تلقائيًا حسب القسم) ← بيانات المريض (الاسم، الجوال، البريد، العمر، الجنس، وصف الأعراض) ← مراجعة نهائية وتأكيد، مع تحقق كامل من صحة البيانات.
- نموذج تواصل يخزن الرسائل في قاعدة البيانات ويحاول إرسال بريد إشعار للإدارة.
- تذييل (Footer) موحد في كل الصفحات: شعار ووصف، روابط سريعة، الأقسام الطبية، معلومات التواصل، وسطر حقوق النشر مع سنة تتحدث تلقائيًا وروابط "سياسة الخصوصية" و"الشروط والأحكام".
- جدول مواعيد عمل ومواعيد الطوارئ، قسم آراء المرضى، وأسئلة شائعة تفصيلية عن الحجز والإلغاء والتأمين.
- حركات ظهور سلسة للعناصر عند التمرير (Scroll Reveal) وحركات نبض لطيفة على عناصر الهيرو.
- تصميم متجاوب بالكامل مع الجوال (قائمة جانبية، شبكات مرنة، جداول قابلة للتمرير).
- حماية أساسية عند التوصيل بقاعدة البيانات: تنظيف المدخلات، Prepared Statements عبر PDO، تشفير كلمات المرور بـ password_hash، تحقق من صحة البيانات في الواجهة (JS) والخلفية (PHP) معًا.

## 🧩 المتوفر جاهزًا للتوسعة (غير مُفعّل افتراضيًا)

- إرسال بريد فعلي (`contact.php` يستخدم `mail()` وتحتاج إعداد SMTP على الاستضافة، أو استبدالها بمكتبة PHPMailer).
- لوحة تحكم للإدارة لعرض المواعيد المحجوزة والرسائل الواردة (الجداول جاهزة في قاعدة البيانات: `appointments`, `contact_messages`).
- ربط تسجيل الدخول/التسجيل بقاعدة البيانات الحقيقية بدلاً من المتصفح فقط: الكود جاهز ومعلّق بعنوان "TODO للاستضافة" داخل `js/auth.js`.
- استرجاع كلمة المرور عبر البريد (الرابط موجود في الواجهة فقط دون خلفية برمجية حاليًا).
- تذكير تلقائي بالموعد عبر SMS/بريد إلكتروني (يتطلب ربط بمزود رسائل خارجي).

## ⚙️ التشغيل خطوة بخطوة (المرحلة 2 — بعد الاستضافة)

1. **متطلبات الخادم**: PHP 8+ و MySQL/MariaDB، عبر XAMPP أو Laragon محليًا أو أي استضافة تدعم PHP.
2. انسخ مجلد `hospital-booking` بالكامل داخل مجلد الخادم (مثال: `htdocs` في XAMPP).
3. استورد قاعدة البيانات:
   ```
   mysql -u root -p < php/database.sql
   ```
   أو من phpMyAdmin: أنشئ قاعدة بيانات ثم استورد ملف `php/database.sql`.
4. افتح `php/config.php` وعدّل:
   ```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'medicare_db');
   define('DB_USER', 'اسم_المستخدم');
   define('DB_PASS', 'كلمة_المرور');
   ```
5. لتفعيل تسجيل الدخول عبر قاعدة البيانات الحقيقية بدل المتصفح فقط، افتح `js/auth.js` وفعّل الأسطر المعلّقة تحت عنوان "TODO للاستضافة" في كل من دالتي تسجيل الدخول والتسجيل.
6. افتح الموقع عبر: `http://localhost/hospital-booking/index.html`

## 🎨 التخصيص السريع

- **الألوان**: عدّل قيم المتغيرات داخل `:root` و `[data-theme="..."]` في أعلى ملف `css/style.css`.
- **النصوص والترجمة**: كل النصوص القابلة للترجمة موجودة في كائن `I18N` داخل `js/lang.js` (قسم `ar` وقسم `en`).
- **الأقسام والأطباء**: عدّل الأقسام والأطباء في `js/appointment.js` (كائن `DOCTORS_BY_DEPT`) وفي `php/database.sql` لتحديثها بعد الاستضافة أيضًا.
- **الشعار**: استبدل الرمز `✚` داخل `.brand-mark` بشعار صورة `<img>` إذا رغبت.

---

## 🔒 توصيات أمنية قبل النشر الفعلي

- استخدم HTTPS دائمًا لحماية البيانات الطبية الحساسة.
- غيّر بيانات الاتصال الافتراضية في `config.php` ولا ترفعها لأي مستودع عام بدون حماية.
- فعّل `session.cookie_httponly` و`session.cookie_secure` في إعدادات PHP.
- أضف حماية CSRF وسجل تدقيق (Audit Log) لأي وصول لسجلات المرضى الطبية.
